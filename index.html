To get the big picture Javaflame

  • Simple general debugging tool
  • Easy and fast to change (no configuration files)
  • Works on any function (including libraries)
  • Does not require code changes (no annotations)
  • Works locally, offline and need no tool to visualize data
  • Captures all argument values
  • See them all together in a single place

Why/Where to use it

  • Not on production
  • Not for measuring performance (unless you have a big error margin)
  • Run locally, on your dev setup.
  • No external dependencies.

To use it

  • Download the agent
  • Add to the JVM command line:
  • Example:
  • Game: Multiples of 3 print Fizz, of 5 print Buzz
  • java -javaagent:javaflame.jar -jar fizzBuzz.jar 40
  • dataForGraph.map(d => d.thread)
  • search(s => s.name.includes("buzz"))

Too much data - Filter expressions

  • Cake Factory demo
  • -javaagent:javaflame.jar=filter:com.example.cakeFactory

Filter expressions

  • Default: Qualified name contains
  • filter:cakeFactory
  • Method matcher
  • filter:com.example.cakeFactory.CakeService#sellCake
  • Conditional match expression
  • filter:(example&&cakeFactory)||(com.foobar)
  • filter:(!org.springframework)&&(!org.hibernate)

Intercept third party (postgres sql)

  • filter:org.postgresql.jdbc.PgConnection#prepareStatement
  • filter:(com.example.cakeFactory)||(org.postgresql.jdbc.PgConnection#prepareStatement)

Ignore calls between two functions

  • filter:(com.example.cakeFactory.CakeController#buyCake)||(com.example.cakeFactory.NationalCakeRegistryService#printResult)

Intercept calls instead

  • "-javaagent:javaflame.jar=intercept:FILTER>METHOD"
